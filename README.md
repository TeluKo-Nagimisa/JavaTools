# EV to SP Calcurater

ポケモンSVで努力値（EV）を振ってからポケモンチャンピオンズへ遠征させることを想定し，目標SPに対して，SV側でのEV投入量と投入順を最適化するJava製コンソールアプリです．

## 概要

このアプリは，ポケモンチャンピオンズ側で必要な目標SPを入力すると，SV側でどのようにEVを投入すれば総コストを小さくできるかを探索します．

本アプリでは，以下の前提で計算します．

- SVでは10EV単位で投入する
- 1能力に割り振れるEV上限は252
- 1個体全体に割り振れるEV上限は510
- 252や510の上限を超えた分は切り捨てる
- EVからSPへの換算は `SP = floor((EV + 4) / 8)` とする
- チャンピオンズ側で不足SPを補う場合，1SPあたりコスト10とする
- SV側で10EVを投入するコストを1とする
- 総コストは `SV側コスト + チャンピオンズ側コスト` とする

## 作成目的

このプロジェクトは，以下の2つを目的として作成しています．

1. ポケモンSVからポケモンチャンピオンズへ遠征させる個体の育成コストを最適化すること
2. Javaの学習と，今後のWebアプリ開発に向けた基礎経験を積むこと

現時点ではコンソールアプリとして実装していますが，将来的にはWebアプリ化を想定しています．

## 主な機能

## 主な機能

- 6能力分の目標SP入力
- 目標SPの入力チェック
  - 1能力あたり最大32SP
  - 合計最大66SP
  - 目標SPが1以上の能力数は3〜6
- 目標SPが1以上の能力だけを探索対象にする
- 目標SPから探索上限EVを自動算出
- 投入希望量を自動生成
- 投入順を全通り生成
  - 有効能力数が4なら4!通り
  - 有効能力数が6なら6!通り
- 順番付き・10EV投入型ルールで最終EVを計算
- 換算SP，不足SP，SV側コスト，チャンピオンズ側コスト，総コストを算出
- 最小総コストの候補を複数表示


## 計算ルール

### EVからSPへの換算

```text
SP = floor((EV + 4) / 8)
```

例：
```text
EV = 252
SP = floor((252 + 4) / 8)
   = floor(256 / 8)
   = 32
```

## 目標SPから探索上限EVを求める式

目標SPを s とすると，そのSPに届くための最小EVは以下です．
```text
EV_min = 8s- 4
```
ただし，SVでは10EV単位で投入するため，10の倍数に切り上げます．
```text
探索上限EV = ceil(EV_min / 10) * 10
```
最大値は260です．
これは，252EVを作るために260EV分の投入が必要になる場合があるためです．

例：
```text
目標SP = 32
EV_min = 8 * 32- 4 = 252
探索上限EV = 260
```

## 順番付き・10EV投入型ルール

このアプリでは，EV投入を以下のように扱います．
	1.	各能力に対して10EV単位で投入を試みる
	2.	1能力上限252を超えた分は切り捨てる
	3.	全体上限510を超えた分も切り捨てる
	4.	最終EVは投入順によって変化する

例：
```text
投入希望量：
260, 100, 160

投入順：
1 → 2 → 3
```
計算：
```text
1番目：260を投入
→ 1能力上限252のため，実際に入るのは252
→ 8EVは切り捨て

2番目：100を投入
→ そのまま100入る

3番目：160を投入
→ 全体上限510まで残りは158
→ 実際に入るのは158
→ 2EVは切り捨て
```
最終EV：
```text
252, 100, 158
```

## 入力仕様

プログラム実行後，6能力分の目標SPを以下の順で入力します．
```text
H A B C D S
```
例：
```text
1 32 1 32 0 0
```
この場合，目標SPが1以上の能力だけが探索対象になります．
```text
H = 1
A = 32
B = 1
C = 32
D = 0
S = 0
```
有効能力数は4です．

## 入力チェック

以下の場合はエラーになります．

1能力が32SPを超える場合
```text
33 10 10 10 0 0
```
合計SPが66を超える場合
```text
32 32 32 0 0 0
```
目標SPが1以上の能力数が3未満の場合
```text
32 32 0 0 0 0
```

## プロジェクト構成

主なクラス構成は以下です．
```text
Main
TargetSp
EvSimulator
SimulationResult
SimulationCandidate
Optimizer
OptimizationResult
```
### Main

コンソール入力を受け取り，Optimizerを呼び出し，結果を表示するクラスです．

### TargetSp

目標SPを保持し，入力値の検証を行うクラスです．

主な役割：
- 6能力分の目標SPを保持
- 1能力32SP以下の検証
- 合計66SP以下の検証
- 目標SPが1以上の能力数が3〜6であることの検証
- 有効能力indexの取得

### EvSimulator

順番付き・10EV投入型ルールに基づいて，1つの候補を評価するクラスです．

主な役割：
- 投入希望量と投入順から最終EVを計算
- EVからSPへ換算
- 不足SPを計算
- SV側コスト，チャンピオンズ側コスト，総コストを計算

### SimulationResult

1つの候補に対する計算結果を保持するクラスです．

保持する情報：
- 最終EV
- 換算SP
- 不足SP
- SV側コスト
- チャンピオンズ側コスト
- 総コスト

### SimulationCandidate

投入希望量，投入順，計算結果をまとめて保持するクラスです．

### Optimizer

目標SPに対して，投入希望量と投入順を探索し，最小総コストの候補を求めるクラスです．

主な役割：
- 目標SPから探索上限EVを算出
- 投入希望量候補を生成
- 投入順を全通り生成
- 各候補をEvSimulatorで評価
- 最小総コストの候補を保持

### OptimizationResult

Optimizerの探索結果を保持するクラスです．

保持する情報：
- 最小総コスト
- 最適候補一覧
- 探索上限EV
- 探索件数
